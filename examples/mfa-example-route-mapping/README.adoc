= Route-Mapping MFA Example
:toc: left
:toclevels: 2
:icons: font
:source-highlighter: rouge
:doctype: article

A working example of an MFA (Micro-Frontend Adapter) that combines
_path-segment rewriting_ with _payload shaping_: the MFA accepts requests
on paths that include a language-segment (`dutch`) inserted after the API
version prefix (e.g. `/v1/dutch/familyties`), strips that segment before
forwarding to the Family Ties microservice, and translates Dutch field
names in the request and response bodies.

This example is derived from the payload-shaping example.
The only behavioural additions are:

. A `RegexRoutingStrategy` bean that rewrites the request path from
  `/v1/dutch/…` to `/v1/…` before forwarding.
. A `RouteMappingProperties` `@ConfigurationProperties` class that carries
  the regex pattern and replacement string from YAML configuration.
. Updated MFE service paths and proxy configuration to call
  `/v1/dutch/…` endpoints.

The microservice is unchanged.
All field-name transformers from the payload-shaping example are reused
without modification.

== Architecture

[source,text]
----
MFE (Dutch fields, /v1/dutch/… paths)
      │
      ▼ RegexRoutingStrategy: /v1/dutch/… → /v1/…
      ▼ DutchToEnglishRequestTransformer
   MFA proxy
      │ EnglishToDutchResponseTransformer
      ▼
MS (English fields, /v1/familyties/… paths)
----

The `RegexRoutingStrategy` and the two transformer beans are activated
exclusively by the `route-mapping` Spring profile.
No MFA framework code was modified; the beans are injected via the SPI.

== Prerequisites

* Java 21
* Node.js 22 / npm 11
* Docker & Docker Compose (for the containerised run)

== Running Locally (without Docker)

=== 1. Start the Microservice

[source,bash]
----
cd ms
export H2_PASSWORD=""
./gradlew bootRun
----

The MS starts on `http://localhost:8081`.

=== 2. Start the MFA (route-mapping profile)

[source,bash]
----
cd mfa
./gradlew bootRun -Dspring.profiles.active=route-mapping
----

The MFA starts on `http://localhost:8080` and forwards requests to `http://localhost:8081`.

=== 3. Start the MFE (dev server)

[source,bash]
----
cd mfe
npm install
npm start
----

The Angular dev server starts on `http://localhost:4200` and proxies API calls to `http://localhost:8080`.

Open `http://localhost:4200` in a browser.

== Running with Docker Compose

[source,bash]
----
cd examples/mfa-example-route-mapping
docker compose up --build
----

This builds and starts all three services in the correct order:

[cols="1,1,2", options="header"]
|===
|Service |Port |Description
|`ms`    |8081 |Family Ties microservice (H2 in-memory)
|`mfa`   |8080 |MFA route-rewriting + field-name-translating proxy (route-mapping profile)
|`mfe`   |4200 |Angular SPA with Dutch UI, served by nginx
|===

Open `http://localhost:4200` once all health checks pass.

To stop and remove volumes:

[source,bash]
----
docker compose down -v
----

=== Running offline (pre-built artifacts)

Use `docker-compose.prebuilt.yml` when Docker cannot reach external registries or package repositories during the build (e.g., behind a corporate firewall).
Pre-build all three services locally, then Docker copies the artifacts instead of running Gradle or npm.

[source,bash]
----
cd examples/mfa-example-route-mapping

# Build ms
cd ms  && ./gradlew :app:bootJar -x test --no-daemon && cd ..

# Build mfa
cd mfa && ./gradlew bootJar -x test --no-daemon       && cd ..

# Build mfe
cd mfe && npm ci && npm run build                      && cd ..

# Start the stack using pre-built artifacts
docker compose -f docker-compose.prebuilt.yml up --build
----

Docker will abort immediately with a `COPY` error if an artifact is missing -- this is intentional (fail fast).

== Field-Name Mapping

[cols="1,1", options="header"]
|===
|Dutch (MFE / MFA surface)
|English (MS surface)

|`voornaam`
|`firstName`

|`achternaam`
|`lastName`

|`vanVoornaam`
|`fromFirstName`

|`vanAchternaam`
|`fromLastName`

|`naarVoornaam`
|`toFirstName`

|`naarAchternaam`
|`toLastName`

|`soort`
|`type`

|`vanPersoon`
|`fromPerson`

|`naarPersoon`
|`toPerson`
|===

Field _values_ are not translated; only JSON key names are renamed.

== route-mapping Profile — What Is Disabled

Under `spring.profiles.active=route-mapping` the following beans are excluded:

* `OidcProviderAdapter` — no OIDC discovery, no JWT validation
* `AuthorizationServiceAdapter` — no SecService token swap
* `AuthenticationApplicationService` / `AuthenticationController` — no login/logout endpoints
* `ReferenceDataEventConsumer` / `KafkaConfiguration` — no Kafka
* `WebClientConfiguration` — replaced by `RouteMappingWebClientConfiguration` (plain HTTP `WebClient`)
